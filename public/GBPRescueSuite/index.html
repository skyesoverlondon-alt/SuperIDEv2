Workflow

Recommended Flow

  1. Start in Intake to define the case and capture the operator ground truth.
  2. Move to Evidence to close proof gaps and document everything that supports reinstatement.
  3. Use Appeals to shape the narrative and escalate with a clean packet.
  4. Send from Outreach, then keep the live follow-up cycle active in Monitoring.
  5. Use SkyeMail, SkyeChat, Neural Space Pro, AE Flow, and SkyeDrive as the execution and support lanes around the five rescue apps.
Persistence

Shared Runtime, Not A Sidecar

The GBP platform now saves case state through the main runtime app-record rail and falls back locally only when the operator is offline or unsigned. This suite page is only a launcher hub over that same runtime.
