Architecture Summary

  • Frontend: Browser-based application hosted on Netlify.
  • Policy/API layer: Netlify Functions enforce user/session controls, token constraints, and auditable actions.
  • Privileged operations layer: Cloudflare Worker performs signed server-to-server operations (push, deploy, export).
  • Persistence: PostgreSQL (Neon) stores organizations, workspaces, audit records, and app event records.
Model call path: Client request → Netlify function gate → policy/auth checks + audit logging → upstream AI gateway call from server context.

Security & Governance Controls

  • Server-side credential model (no provider secrets exposed to browser clients).
  • Signed service-to-service requests between policy layer and privileged runner.
  • Org/workspace access checks applied on export, deploy, and push operations.
  • Token model supports scope restrictions, expiry, lock constraints, and revocation.
  • Comprehensive audit logging for generation, export, push, and deploy actions.
